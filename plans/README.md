# Plans

Complex work is tracked under `plans/YYYY.MM/DD/<title-slug>/`.

Each packet starts with `plan.md`. Add `implementation.md` only after work has
occurred and add `test.md` when verification evidence is useful. Keep
[`open.md`](open.md) and [`closed.md`](closed.md) synchronized with packet
status. See [WORKFLOW.md](../WORKFLOW.md) for the full lifecycle.
